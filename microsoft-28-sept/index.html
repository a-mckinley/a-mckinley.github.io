LLM Application Taxonomy

  • LLM Chat
  • AI Enabled Search
  • RAG
  • RAG + Agents
  • Autonomous Agents

Taxonomy: 1 - LLM Chat

  • Most basic use
  • Simply chat with an LLM like ChatGPT
  • “Coroporate” ChatGPT
    • Prompts/context never leave Azure with Azure OpenAI
  • Minimal infrastructure required

Taxonomy: 2 - AI Enabled Search

  • Leverage “embeddings” from encoder models
  • Semantic search with vector ANN
    • Topics/themes/meaning as opposed to keywords
  • Likely requires vector database or database with vector ANN/index
  • Hybrid search may improve quality

Taxonomy: 3 - Retreival Augmented Generation (RAG)

  • Combine search (semantic or otherwise) with LLMs on output
  • Summarise/condense/extract from retrieved context
  • Important technique for “grounding” LLM with knowledge base (e.g. sharepoint docs/website content)

Taxonomy: 4 - Retreival Augmented Generation (RAG) + AI Agents

  • Combine RAG with task specific LLM based “agent”
  • Agent being a prompt chain/flow that completes a specific task
  • E.g. generate new content about a requested subject based on existing

Taxonomy: 5 - Autonomous Agents

  • Complex task completion with tuned LLM agents
  • Access to “tools”
    • e.g web search
  • Potentially non-derministic execution
    • Allow LLM agent to make decisions about sub-goals

Prompt Engineering and Other “Tricks”

Prompt Engineering?

  • LLMs have little inherent idea about success
  • Giving them some hints produce better results
  • “Lets think about this step by step”
  • “You are an expert in X”

Why Is It Important?

  • PE and other “tricks” can drastically improve performance
  • Case study shortly

What Kind of “Tricks”?

  • Chain of Thought
    • Zero Shot - “Lets Think about this step by step”
    • Few Shot - Examples of similar problems (e.g. calculations)
  • Tree of Thought
    • Create multiple intermediate solutions, promote plausible
  • Self critique/reflection
    • Add guidlines, ask the LLM to review its response based on guidlines
  • ReAct
    • Reason + Act (can use tools and create goals)
  • Chain of Density
    • A specific prompt format, looped to incremental add missing details
  • Many others…

Case Study

  • Collect evidence about specific entities from the web
  • Instruct LLM to:
    • Evaluate search results for “entity”
    • Determine if a specific attribute appears to be true about the entity
    • Return results to user about confidence of the attribute
  • GPT3.5
    • Poor accuracy
  • GPT4
    • Better accuracy
  • GPT4 plus:
    • Extensive and verbose prompt
    • Clear specific guidelines
    • Self reflection step on adherence to guildlines
    • Very good accuracy
  • Interestingly, GPT3.5 did not really benefit in this case from the reflection step
